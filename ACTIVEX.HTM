///////////////////////////////////////////////////////////////////////////////
// ACTIVEX
///////////////////////////////////////////////////////////////////////////////


Embeds an ActiveX control into a form, providing access to all its functionality.


Control creation:

*** Standard syntax:

   @ <row>, <col> ACTIVEX <name> ;
      PROGID <progid> ;
      [ OBJ <obj> ] ;
      [ <OF, PARENT> <parent> ] ;
      [ WIDTH <width> ] ;
      [ HEIGHT <height> ] ;
      [ <notabstop: NOTABSTOP> ] ;
      [ <disabled: DISABLED> ] ;
      [ SUBCLASS <subclass> ]

   Is translated by the preprocessor to this OOP syntax:

   [ <obj> := ] _OOHG_SelectSubClass( TActiveX(), [ <subclass>() ] ): ;
      Define( <(name)>, <(parent)>, <col>, <row>, <width>, <height>, ;
      <progid>, <.notabstop.>, <.disabled.> )

*** Alternative syntax:

   DEFINE ACTIVEX <name>
      PROGID <progid>
      [ COL <col> ]
      [ DISABLED <.T. | .F.> ]
      [ HEIGHT <height> ]
      [ OBJECT <obj> ]
      [ PARENT <parent> ]
      [ ROW <row> ]
      [ SUBCLASS <subclass> ]
      [ TABSTOP <.T. | .F.> ]
      [ WIDTH <width> ]
   END ACTIVEX

   Is translated by the preprocessor to this OOP syntax:

   _OOHG_SelectSubClass( TActiveX(), <subclass>(), [ { |_o_| <obj> := _o_ } ] ): ;
      Define( <(name)>, <(parent)>, <col>, <row>, <width>, <height>, ;
      <progid>, ! <tabstop>, ! <disabled> )

*** Alternate OOP syntax:

   [ <oObj> := ] TActiveX():Define( cName, [ cParent ], [ nCol ], [ nRow ], ;
      [ nWidth ], [ nHeight ], cProgID, [ lNoTabstop ], [ lDisabled ] )


Notes on clauses:

   1. PROGID identifies an ActiveX control. See Active Template Library.
   2. OBJ and OBJECT set the variable <obj> with a reference to the control.
      When using OBJECT, the variable must be previously declared.
      When using OBJ, if the variable does not exist, it's assumed PRIVATE.
   3. PARENT declares which window is the parent (owner) of the control.
      Can be ommited only when used inside a DEFINE WINDOW / END WINDOW structure.
   4. ROW and COL define the position of the control inside it's parent client area.
   5. WIDTH and HEIGHT define the dimension of the control.
   6. TABSTOP and NOTABSTOP specify if the control can/can't be focused using TAB key.
      Default is TABSTOP.
   7. DISABLED specifies that the control can't receive mouse clicks and key presses.
      Default is ENABLED.
   8. SUBCLASS indicates that the control must be an instance of the specified class,
      instead of the default TActiveX class.


Notes on parameters:

   1. <row>, <col>, <width> and <height> must be numbers.
   2. <name> and <parent> must be valid identifiers enclosed or not within quotes.
   3. <progid> must be a string.
   4. <obj> and <subclass> must be valid identifiers not enclosed within quotes.
